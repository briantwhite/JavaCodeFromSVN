Running VGL - Virtual Genetics Lab

To run VGL Admin simply double-click on the VGLAdmin.exe file. Note: Be sure to have the images and Help folder in the same path as the VGLAdmin.exe file.
When VGL Admin launches, you will see this window:


How To Open an Existing Problem

You can use this feature to load up previously saved problems that you might have been created. To open a problem you can either select "Open Problem" from the drop-down "File" menu or simply click on the "Open Problem" button on the toolbar. A dialog box will open up to assist you in selecting a file. Browse to the directory in which the problem file is stored. Select the file and click on the "Open" button. VGL will read the file and will open up the problem and bring all the values you last left it in before you saved and closed the problem.


How To Save the Problem

You can use this feature to save your current work to a file. To save your work you can either select "Save" from the drop-down "File" menu or simply click on the "Save" button on the toolbar. A dialog box will open up to assist you in saving your work to a file in the directory of your choice. Browse to the directory in which you want to save your work. In the "File Name" field enter the name of the file you want you want to save your work as. Then click on the "Save" button. This will create and save a file with the name as specified in the field and will store your current work in it.


How to Set the values to default

You can set all the value back to default setting by select "Clear" from the drop-down Edit menu or simply click on the "Clear" button on the toolbar. It will set the chance of simple dominance to 50, chance of autosomal to 50, chance of XX (male) and XY (female) to 50, uncheck the "Practice Mode" and disable the email function.


How to Set limits on the problem:

The three sliders set the limits on the genetic models that can appear in a problem. Once a problem is set, the model, phenotypes, and genotypes of the starting population are generated randomly for each run.
All the problems in VGL involve genetic models with one gene that has two alleles. Based on this, there are several features that can vary:

  • The interaction between the alleles; this can be either:
    • Simple Dominance: (Models 1, 3, & 5) The heterozygote has the same phenotype as the dominant homozygote. That is, with two alleles A and a:
      • AA = tall
      • Aa = tall
      • aa = short
    • Incomplete Dominance: (Models 2, 4, & 6) The heterozygote has a different phenotype than either homozygote. In nature, this is usually intermediate; in VGL it need not be.
      • AA = tall
      • Aa = medium
      • aa = short
  • Whether the character is sex-linked or not; this can be either:
    • Not sex-linked - the gene for the character is carried on an autosome so it is inherited identically in both sexes. (Models 1 & 2)
    • Sex-linked - the gene for the character is located on a sex-chromosome so it is inherited differently in different sexes. This has two forms:
      • XX/XY Females are XX; males are XY (Models 3 & 4). Here, Y carries no genes except those needed to make the organism male.
      • ZZ/ZW Females are ZW; males are ZZ (Models 5 & 6). Here, W carries no genes except those needed to make the organism female.

This leads to six possible genetic models.

Slider #1: Controls the chance that the student will get a model based on simple or incomplete dominance.

  • All the way to the left (0%) means that all the problems will involve incomplete dominance. Models 2, 4, and 6.
  • All the way to the right (100%) means that all the problems will involve simple dominance. Models 1, 3, and 5.
  • In between means that some problems will involve simple dominance while others will involve incomplete dominance.
Slider #2: Controls the chance that the student will get a model based on sex linkage or not.
  • All the way to the left (0%) means that all the problems will involve sex-linkage. Models 3, 4, 5, and 6.
  • All the way to the right (100%) means that all the problems will involve autosomal inheritance. Models 1 and 2. Note that, in this setting, the mode of sex linkage is irrelvant so slider 3 is grayed out.
  • In between means that some problems will involve sex-linkage while others will not.
Slider #3: Controls the chance that if the model involves sex-linkage, the student will get a model based on XX/XY or ZW/ZZ sex-linkage.
  • All the way to the left (0%) means that all the problems will involve ZZ/ZW sex-linkage. Models 4 and 6.
  • All the way to the right (100%) means that all the problems will involve XX/XY sex-linkage. Models 3 and 5.
  • In between means that some problems will involve XX/XY sex-linkage while others will involve ZZ/ZW sex-linkage.

Practice Mode

If you check the "Practice Mode" box at the bottom of the window, Practice mode will be available to the students.
In Practice mode, the students can choose to see the underlying genetic model and the genotypes of all the organisms in their problem. This is useful for students who are learning to use the program. In Practice mode, students can also choose not to see the model and genotypes by clicking a button when they open their problem.
If you do not check the Practice Mode box, the students will not be able to see the model and genotypes under any circumstances. This is the mode the program is supposed to be run in.
